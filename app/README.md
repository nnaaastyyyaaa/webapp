## Опис застосунку згідно з варіантом(V3):

Task Tracker — сервіс для відстеження задач.

    Об’єкт задачі має наступні поля:
    - id
    - title
    - status
    - created_at

    API сервісу складається з 3 ендпоінтів:
    - GET /tasks — вивести усі задачі (id, title, status, created_at)
    - POST /tasks (title) — створити нову задачу
    - POST /tasks/<id>/done — змінити статус задачі на виконано

### Порт застосунку та конфігурація(V5 та V2):

Конфігураційний файл за шляхом /etc/mywebapp/config.env.
Порт: 5200

### База даних(V2):

PostgreSQL

## Застосунок та його призначення

Mywebapp - сервіс для відстеження задач. Допомагає прослідковувати виконані та невиконані задачі

## API застосунку

### GET /health/alive

Завжди повертає статус 200(ОК), якщо сервер працює.

```bash
curl http://0.0.0.0:80/health/alive
```

### GET /health/ready

Поверне статус 200(ОК), якщо сервіс успішно підключився до бази даних, інакше - поверне повідомлення про причину помилки.

```bash
curl http://0.0.0.0:80/health/ready
```

### GET /

Поверне список усіх ендпоінтів, які існують в межах сервісу. Приймає запит лише з заголовком 'Accept: text/html'.

```bash
curl http://0.0.0.0:80/ -H 'Accept: text/html'
```

### GET /tasks

Поверне список усіх створених задач

```bash
curl http://0.0.0.0:80/tasks
```

### POST /tasks

Дозволяє створити нову задачу

```bash
curl -X POST http://0.0.0.0:80/tasks -d '{"title":"to do homework"}' -H 'Content-Type: application/json'
```

### POST /tasks/:id/done

Дозволяє позначити відповідну задачу виконаною

```bash
curl -X POST http://0.0.0.0:80/tasks/2/done -H 'Content-Type: application/json'
```

За замовчуванням сервіс повертає відповідь у вигляді json-формату, проте можна додати заголовок 'Accept : text/html', тоді сервіс віддасть html-сторінку

```bash
curl -X POST http://0.0.0.0:80/tasks/2/done -H 'Accept: text/html' -H 'Content-Type: application/json'
```

## Запуск застосунку

Після успішного встановлення застосунку, systemd автоматично керує запуском сервісу за допомогою [systemd socket activation](https://ilmanzo.github.io/post/systemd-socket-activated-services/) та reverse proxy. Застосунок запускається від системного користувача app.

Якщо до застосунку було внесено якісь зміни, то для того , щоб вони почали діяти потрібно перезапустити сервіси

```bash
sudo systemctl restart mywebapp
sudo systemctl reload nginx
```

Якщо було змінено структуру бази даних

```bash
npx prisma migrate dev --name added_something_new
sudo systemctl restart mywebapp
sudo systemctl reload nginx
```

## Розгортання застосунку mywebapp

### Встановлення операційної системи Linux

1. Для початку потрібно встановити образ [Ubuntu 24](https://ubuntu.com/download/server#manual-install-tab), а також програму [VirtualBox](https://www.virtualbox.org/) для створення віртуальної машини.

2. Після успішного завантаження відкрийте програму VirtualBox та у верхньому лівому куті натисніть "New".

3. У вікні , яке зʼявилось обовʼязково задайте такі налаштування:
   - VM Name: Linux
   - User Name: ubuntu
   - Задайте пароль
   - Host name: Linux
   - Base memory: 6000 MB
   - Number of CPUs: 2
   - Disk Size: 60 GB

4. Натисність кнопку Finish.

   Після запуску скрипта розгортання почнеться створення віртуальної машини.
   Це може зайняти від 5 до 30 хвилин залежно від ресурсів системи.
   Коли процес завершиться, ви побачите командний рядок термінала з промптом login:.
   У промпті login напишіть ubuntu. Далі система попросить ввести пароль, важливо ввести саме той, який ви вкаазали при створенні віртуальної машини. Також потрібно зауважити, що під час введення паролю ви не будете бачити літери, які вводяться в терміналі.

5. Після успішного встановлення сервера ubuntu потрібно встановити графічний інтерфейс. Для цього по черзі запускайте команди:

   ```bash
   sudo apt update
   sudo apt install ubuntu-desktop -y
   ```

   Встановлення графічної версії також може тривати від 5 до 30 хвилин. Після успішного встановлення ви побачите повідомлення про це і активний командний рядок.

6. Коли графічну версію буде встановлено, потрібно перезавантажити систему, застосуйте наступну команду:

   ```bash
   sudo reboot
   ```

7. Після перезапуску ви побачите вікно графічного інтерфейсу з можливістю увійти в систему під створеним раніше користувачем ubuntu, що і потрібно зробити.

8. Відкрийте термінал та запустіть в ньому таку команду

   ```bash
   sudo git clone https://github.com/nnaaastyyyaaa/mywebapp.git /opt/mywebapp
   ```

9. Зробіть файл розгортання застосунку виконуваним , використовуючи наступну команду

   ```bash
   sudo chmod +x /opt/mywebapp/run-mywebapp.sh
   ```

10. Запустіть скрипт розгортання застосунку

    ```bash
    sudo /opt/mywebapp/run-mywebapp.sh
    ```

11. Після успішного встановлення всіх залежностей та сервісів необхідно перезапустити систему, щоб всі зміни корректно почали працювати

    ```bash
    sudo reboot
    ```

12. На сторінці входу в систему ви побачите можливість увійти під одним з двох користувачів: operator або teacher. Користувач operator має обмежені права: запуск, перевірка статусу, перезапуск, зупинка застосунку mywebapp та перезапуск nginx. Натомість teacher має можливість підвищувати свій статус до root. Оберіть користувача та увійдіть в систему, потрібно буде замінити дефолтний пароль (12345678) на ваш власний.

13. В системі окрім двох вище вказаних користувачів також існує користувач student. На момент створення він не має пароля, тому система при перезапуску не дає увійти в системк під цим користувачем. Також student має можливість ескалації до root, але це також вимагає створення пароля.Тому щоб мати необмежені можливості під користувачем student, треба поставити пароль, для цього відкрийте термінал та введіть таку команду:

    ```bash
    sudo passwd student
    ```

    Після введення цієї команди система попросить вас створити пароль.

14. Тепер після перезавантаження ви можете увійти в систему використовуючи одного з трьох користувачів.

15. Відкриємо термінал та перевіримо статус systemd та nginx

    ```bash
    sudo systemctl status mywebapp.socket
    sudo systemctl status nginx
    ```

Вивід має показувати 'active (running)'

16. Тепер можна важати розгортання застосунку завершеним і тестувати застосунок, використовуючи ендпоінти, вказані в розділі [API застосунку](#api-застосунку)

### Розгортання за допомгою Docker

1. Встановіть [Docker](https://www.docker.com/products/docker-desktop/)

2. Після успішного завантаження запустіть Docker Desktop

3. Відкрийте вікно терміналу та введіть команду

```bash
git clone https://github.com/nnaaastyyyaaa/mywebapp.git
```

4. Перейдіть у папку mywebapp

```bash
cd mywebapp
```

5. Запустіть команду

```bash
docker compose up --build
```

6. Після успішної збірки у вікні браузера або у терміналі можна тестувати [ендпоінти](#api-застосунку)
